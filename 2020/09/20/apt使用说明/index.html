apt使用说明

Posted on 2020-09-20 Edited on 2020-09-21 Views:
本文叙述了apt是为了解决何种问题出现的，apt常用指令，apt具体做了什么。

为什么需要包管理工具

最早在linux系统上安装软件是需要通过源码进行构建的。最初只有.tar.gz 的打包文件，用户必须编译每个他想在 GNU/Linux 上运行的软件。这种方法十分复杂且容易出错，于是就有将这一系列安装过程打包起来的需求，就产生了deb包。deb包中的内容可以看该文章如何制作deb包，其中列出了deb包中的内容。

在有了deb包之后，安装软件似乎变得简单了许多，但是新的问题又出现了，在deb包中有一个Depends项，这是该deb包安装所需要的前提条件，如果这些前提条件未能满足，则deb包无法正常安装（在使用dpkg安装deb包时，依赖错误是很常见的一种）。并且这个依赖关系可能是一个很大的树状关系。而apt就是提供一个能够解决这些依赖关系的高级包管理工具

软件依赖的解决思路

apt就是通过一个数据库来记录各种软件安装包的依赖关系，这样每一个包都可以生成一个依赖关系树。同时apt还记录着本地下载的软件列表。在安装软件时，apt会检查该软件依赖树，并与本地软件列表进行对比，将其所需要的所有本地没有的依赖一并下载。完成软件的安装。

APT是什么

首先应用debian wiki的原话

  • Advanced Package Tool (or APT), the main command-line package manager for Debian and its derivatives. It provides command-line tools for searching, managing and querying information about packages, as well as low-level access to all features provided by the libapt-pkg and libapt-inst libraries which higher-level package managers can depend upon.

意思就是APT是debian以及其衍生产品（ubuntu等）的主要命令行包管理工具。它提供了命令行工具，用于搜索，管理和查询有关软件包的信息。同时提供更高级的包管理器所依赖的libapt-pkg和libapt-inst库提供的所有功能的低级访问。

这里需要注意大写的APT是Debian使用的一组高级打包工具，用来管理其打包系统。不要将它与apt命令混淆，这是不一样的。有多种工具可以与APT交互，并允许您在基于Debian的Linux发行版中安装，删除和管理软件包。apt-get是一种广泛使用的命令行工具。另一个流行的工具是 同时具有GUI和命令行选项的Aptitude

apt常用命令

  • 更新软件清单：sudo apt update
  • 升级软件包：sudo apt upgrade
  • 升级软件包，升级前先删除需要更新软件包：sudo apt full-upgrade
  • 安装软件包：sudo apt install <package_1> <package_2> <package_3>
  • 更新指定的软件：sudo apt update <package_name>
  • 显示软件包具体信息,例如：版本号，安装大小，依赖关系等等：sudo apt show <package_name>
  • 删除软件包：sudo apt remove <package_name>
  • 清理不再使用的依赖和库文件: sudo apt autoremove
  • 移除软件包及配置文件: sudo apt purge <package_name>
  • 查找软件包： sudo apt search <keyword>
  • 列出所有{已安装 |可升级|所有版本}的包：apt list {--installed | --upgradeable | --all-version}

apt-get常用指令

  • 搜索包：apt-cache search <package_name>
  • 获取包的相关信息，如说明、大小、版本等：apt-cache show <package_name>
  • 安装包：apt-get install <package_name>
  • 重新安装包：apt-get install <package_name> - - reinstall
  • 修复安装”-f = –fix-broken”：apt-get -f install
  • 删除包：apt-get remove <package_name>
  • 删除包，包括删除配置文件等：apt-get remove <package_name> - - purge`
  • 更新源：apt-get update
  • 更新已安装的包：apt-get upgrade
  • 更新包（更自动处理冲突，有风险）：apt-get dist-upgrade
  • 使用 dselect 升级：apt-get dselect-upgrade
  • 查看包依赖：apt-cache depends <package_name>
  • 是查看该包被哪些包依赖：`apt-cache rdepends ``
  • 安装相关的编译环境：apt-get build-dep <package_name>
  • 下载该包的源代码：apt-get source <package_name>
  • 清理无用的包：apt-get clean && apt-get autoclean
  • 检查是否有损坏的依赖：apt-get check

apt与apt-get区别

apt-get是之前围绕APT最受欢迎的工具。但是它似乎有些过于繁琐，所以apt就出来将apt-get与apt-cache中常用的指令提取出来，形成一个新的方法集。

apt是apt-get和apt-cache命令的子集，为软件包管理提供必要的命令

虽然apt-get未被弃用，但作为普通用户应该更频繁地使用apt

apt的命令行形式还没稳定，所以在编写脚本时更倾向于使用apt-get

apt工作原理（apt-get基本相同）

  1. apt update

该操作就是访问/etc/apt/source.list，以及/etc/apt/source.list.d/*中的所有条目（关于 /etc/apt/sources.list的信息可以参考debian软件源source.list文件格式说明），获取这些条目中的软件信息，并将这些信息放入/var/lib/apt/lists中

我们来看看/var/lib/apt/lists中的东西是什么：

首先我们找到一类名字中有Packages的文件

我们可以看到Packages文件中是很多的存储包信息文档，在这些文档储存了包的依赖关系以及一些其他描述，而依赖树就是通过这些信息生成的。当然它还有很多的其他信息，如校验和等。

  1. apt install <package>

该指令会通过/var/lib/apt/lists获取软件所需要的安装信息，并生成依赖关系树，最后确定安装所需的依赖，并且安装所有未安装的依赖项

  1. apt upgrade

在apt update后，会获取软件的更新信息，apt upgrade会将所有可更新的包进行更新。

  1. apt remove <package>

该指令会移除一个包。但是问题出现了，移除一个包，其安装时附加的依赖项会被保留，但是却又是无用的，这样会造成很大的资源浪费。另一方面，如果直接将所有进行移除，也会出现问题，假设A包与B包同时依赖C包，当删除A包时就不能删除C包，只有A，B都被remove后才能确定这是一个无用的包。

有了这个思路，就可以去思考apt的逻辑。apt对每一个包信息都保存一个依赖计数器与一个用户下载标记，记录其被依赖的数量，当其被依赖的数量降为0，且其不是用户主动下载时，其就可以视为无用的。apt autoremove就是会检查这一类型的包并进行删除。

删除一个包时，会对该包的所有直接依赖的依赖计数器进行减1，并且递归的对直接依赖进行操作（假设依赖A还直接依赖B，若A被视为无用的，B的依赖也减一，依次递归）。

  1. apt autoremove

将所有不被依赖且非用户下载的包进行删除

添加软件源

有时候我们需要下载一些官方软件源中没有的软件，如果还想通过apt的方式进行安装，就需要添加第三方的软件源（前提是该软件有第三方源），之后就可以使用apt进行安装。

  1. 添加apt仓库

添加仓库常用格式

sudo add-apt-repository "deb https://developer.../ubuntu2004/x86_64/ /"

sudo add-apt-repository ppa:bashtop-monitor/bashtop

curl -s -L https://.../xx.list | sudo tee /etc/apt/sources.list.d/xx.list

第一种是添加第三方仓库，"deb https://developer.../ubuntu2004/x86_64/ /"会附加在/etc/apt/sources.list末尾

第二种是添加私人仓库，该方法会在/etc/apt/sources.list.d创建一个文件内容为ppa:bashtop-monitor/bashtop。这种方法ubuntu会自动配置gpg-key，不需要手动添加。

第三种方法是手动修改，可以选择在/etc/apt/sources.list后append条目，也可以在/etc/apt/sources.list.d创建新的文件来保存条目，但是文件必须以.list结尾

  1. 添加秘钥

为了验证第三方软件仓库的真实性，防止安装恶意的软件，第三方的软件仓库都会提供GPG公钥来进行身份验证，防止获取到的是被串改的恶意软件源，apt-key用于管理apt验证软件包的密钥列表，被添加的包视为可信任的。

在添加apt仓库时，需要将该仓库的密钥加入受信任的列表中，不然无法进行update。

添加key常用格式

wget -qO - https://typora.io/linux/public-key.asc | sudo apt-key add -

curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -

sudo apt-key adv --fetch-keys https://developer.../ubuntu2004/x86_64/7fa2af80.pub

注：sudo apt-key add - 中 -代表标准输出，即访问该网址将输出写入key中

同样，apt-key也可以使用修改文件的方法添加

直接将.gpg文件放入/etc/apt/trusted.gpg.d/中也是可以的

删除软件源

删除与添加的方法相似，需要删除repository与gpg-key

apt-add-repository -r <repository> # deb或ppa打头的字符串

apt-key del <keyid> # key的末8位

同样可以用修改文件的方法删除，只要删除对应的.list与.gpg文件即可
